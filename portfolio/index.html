Database And Seed Data

MySQL schema supports organizations, sites, roles, sessions, buildings, floors, rooms, zones, devices, schedules, alarms, trend logs, analytics, optimization runs, firmware jobs, maintenance tickets, and report exports.

  • Engine: MySQL 8 in Docker service db.
  • Driver: mysql2 from the Node API.
  • Seed login: admin / admin for local evidence.

Energy And Optimization

Portfolio-seeded comparison frames baseline load, optimized load, peak interval demand, comfort risk, demand-response state, and operator action.

  • Baseline: 68.0 kWh sampled building load.
  • Optimized: 50.0 kWh sampled building load.
  • Delta: 18.0 kWh projected savings in seeded evidence.

AI Service Model Card

BEMS-ai is documented as advisory support, not a life-safety controller. Final actuator writes remain behind edge-core limits and operator review.

  • Inputs include telemetry, device state, setpoints, schedules, and feedback.
  • Outputs include optimization JSON, savings projections, and recommendations.
  • Controls include confidence/health surfacing and fallback dashboard operation.

Operator Workflow

How The Dashboard Is Reviewed

The BMS portfolio should be inspected like an operations dashboard: start at access, move through building state, inspect room and schedule records, handle alarms, then confirm service health and evidence.

1

Login And Role Context

Open the BMS access page, select Operator, Engineer, or Admin, then submit the seeded credentials to land on the EnergyBuildAI dashboard.

2

Building Review

Confirm building mode, floors monitored, rooms scheduled, zone load, fans, dampers, motors, pump motor, lighting relay bus, and energy heat-map status.

3

Room And Schedule Review

Inspect room mode, setpoint, schedule source, next event, override state, and control intent for critical areas such as the server room.

4

Alarm Triage

Select the highest-priority alarm, review source, reading, threshold, likely cause, impact, and recommended action before acknowledge or clear operations.

5

Optimization Review

Compare baseline and optimized energy state. Treat BEMS-ai output as advisory until live building calibration and safety writeback limits are verified.

6

Health And Evidence

Check API health, watchdog state, Docker service status, CI artifacts, and screenshot evidence before considering the review package complete.
